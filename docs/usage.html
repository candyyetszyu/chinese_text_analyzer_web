Usage Overview: This guide covers both web interface and command-line usage patterns. Choose the method that best fits your workflow.

網頁介面

The web interface provides an intuitive way to analyze Chinese text with real-time results and interactive visualizations.

Starting the Web Server

# Start the web application
python run_web.py

# Or with custom settings
python run_web.py --host 0.0.0.0 --port 8080 --debug

Accessing the Interface

Once started, access the web interface at:

  • Local: http://localhost:5000
  • Network: http://your-ip:5000
  • Production: Configure with nginx/apache
Production Note: For production deployment, use a WSGI server like Gunicorn and configure proper security settings.

Command Line Interface

Use the powerful CLI for batch processing, automation, and integration into workflows.

Basic CLI Usage

# Basic analysis
python run_cli.py --file input.txt

# With output specification
python run_cli.py --file input.txt --output results.json

# Multiple files
python run_cli.py --file file1.txt file2.txt file3.txt

# Directory processing
python run_cli.py --directory ./text_files/ --recursive

CLI Options Reference

Input Options
--file, -fInput file(s)
--directory, -dInput directory
--text, -tDirect text input
--encodingFile encoding (default: utf-8)
--recursiveProcess subdirectories
Output Options
--output, -oOutput file path
--formatOutput format (json/csv/txt)
--visualizeGenerate visualizations
--no-statsSkip basic statistics
--quiet, -qSuppress output messages

Advanced CLI Examples

# Custom analysis with specific features
python run_cli.py --file news.txt \
    --output analysis.json \
    --visualize \
    --sentiment \
    --entities \
    --keywords 20

# Batch processing with configuration
python run_cli.py --directory ./corpus/ \
    --recursive \
    --config custom_config.json \
    --output-dir ./results/ \
    --format csv

# Pipeline processing
python run_cli.py --file input.txt | \
    python post_process.py | \
    python generate_report.py

Batch Processing

Process multiple files efficiently with batch operations and parallel processing.

Batch Scripts

# Process all .txt files in directory
find ./documents -name "*.txt" -exec \
    python run_cli.py --file {} \
    --output ./results/{}.json \;

# Parallel processing with GNU parallel
ls *.txt | parallel -j 4 \
    python run_cli.py --file {} \
    --output results/{.}.json

# Batch with shell script
#!/bin/bash
for file in ./input/*.txt; do
    echo "Processing $file"
    python run_cli.py --file "$file" \
        --output "./output/$(basename "$file" .txt).json"
done

Performance Tips

Optimization Tips
  • Use --parallel for multiple cores
  • Set appropriate --batch-size
  • Use --cache for repeated operations
  • Consider --memory-limit for large files
  • Enable --progress for monitoring
Monitoring Progress
# Progress monitoring
python run_cli.py --directory ./large_corpus/ \
    --progress \
    --log-level INFO \
    --log-file batch.log

Advanced Options

Fine-tune analysis behavior with advanced configuration options.

Analysis Configuration

# Custom dictionary
python run_cli.py --file input.txt \
    --dict-file custom_dict.txt \
    --stopwords custom_stopwords.txt

# Sentiment configuration
python run_cli.py --file input.txt \
    --positive-words pos_words.txt \
    --negative-words neg_words.txt \
    --sentiment-threshold 0.6

# NER configuration
python run_cli.py --file input.txt \
    --ner-model custom_ner_model \
    --entity-types PER,LOC,ORG

Performance Tuning

# Memory optimization
python run_cli.py --file large_file.txt \
    --chunk-size 1000 \
    --memory-limit 2GB \
    --cache-size 500MB

# Parallel processing
python run_cli.py --directory ./files/ \
    --parallel 4 \
    --batch-size 10 \
    --timeout 300

# GPU acceleration (if available)
python run_cli.py --file input.txt \
    --use-gpu \
    --gpu-memory 1GB

Integration Options

Database Integration
# Save to database
python run_cli.py --file input.txt \
    --db-url postgresql://user:pass@host/db \
    --table-name analysis_results

# MongoDB export
python run_cli.py --file input.txt \
    --mongo-url mongodb://localhost:27017 \
    --collection text_analysis
API Integration
# Webhook notifications
python run_cli.py --file input.txt \
    --webhook-url https://api.example.com/notify \
    --webhook-events complete,error

# API export
python run_cli.py --file input.txt \
    --api-endpoint https://api.example.com/results \
    --api-key your_api_key
